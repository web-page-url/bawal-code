Bawal Code is a custom, Hindi-inspired high-level programming language designed to make coding more accessible and expressive. Unlike simple transpilers, it features a fully custom-built compiler stack comprising a recursive-descent parser and a secure runtime interpreter.

The primary goal is to provide a complete development environment ‚Äî from a web-based playground to a standalone NPM Package for CLI execution.

‚öôÔ∏è The Execution Pipeline

When you press "Compile" or run a .bawal file, your code travels through these distinct phases:

1. Lexical Analysis (Lexer)

The raw text is broken down into Tokens. It recognizes keywords (kaam, agar, bol), operators (+, -, ==), and literals while tracking line/column numbers for accurate error reporting.

2. Syntax Analysis (Parser)

Tokens are structured into an Abstract Syntax Tree (AST). Our parser uses Operator Precedence (BODMAS) to ensure expressions like 2 + 3 * 4 result in 14, not 20.

3. Programmatic Execution (Interpreter)

The interpreter traverses the AST node-by-node. It maintains its own Environment (Scope) for variables and functions, executing logic directly using JavaScript's native capabilities but without using dangerous tools like eval().

üõ°Ô∏è Security & Sandboxing

Security is the backbone of Bawal Code. Since it runs in the browser and on user machines via NPM, we implemented strict sandboxing:

  • üö´ No eval(): The language is interpreted, not evaluated as raw JS string.
  • ‚è±Ô∏è Execution Limits: Programs have a 5-second timeout to prevent CPU hang.
  • üîÑ Loop Protection: Infinite loops are caught after 1000 iterations.
  • üîí No Global Access: The interpreter cannot access the window or document objects.
